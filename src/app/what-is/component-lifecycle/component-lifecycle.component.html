What is:
Component Lifecycle

A component goes through a defined series of phases from creation to destruction, known as the component lifecycle:

  1. Creation:
    • ngOnChanges: Invoked when input properties change after initialization. Useful for reacting to external data changes.
    • ngOnInit: Called after the component has been initialized and its view has been created. Perfect for fetching data, subscribing to events, and setting up initial state.
  2. Change Detection:
    • ngDoCheck: Triggered after every change detection cycle. Often used for debugging or performance optimization.
  3. View Updates:
    • ngAfterContentChecked: Called after content children (components projected through ng-content ) have been checked.
    • ngAfterViewInit: Invoked after the component's view (including child components) has been fully initialized. Ideal for accessing or manipulating DOM elements directly.
  4. Changes and Destruction:
    • ngOnChanges: Called again if input properties change again after initialization.
    • ngOnDestroy: Triggered just before the component is destroyed. Used for cleaning up resources like subscriptions, timers, and event listeners.
  5. Additional hooks:
    • constructor: Typically used for injecting dependencies and performing basic setup.
    • ngAfterViewChecked: Similar to ngAfterViewInit , but occurs after change detection has completed.
