Component Lifecycle

Every component in Angular follows a well-defined lifecycle, consisting of various stages from creation to destruction.

Key Stages:
  1. Creation:
    • ngOnChanges() (optional): Triggers when input properties change.
    • ngOnInit(): Invoked after the component receives its initial inputs and the content is available. Ideal for fetching data, setting up subscriptions, or performing computations.
    • ngDoCheck() (optional): Executes after every Change Detection cycle, enabling custom change detection logic.
  2. Change Detection:
    • This stage involves detecting changes in data-bound properties and updating the view if necessary.
    • ngAfterContentChecked() (optional): Called after content is checked, useful for tasks like DOM manipulation based on content changes.
    • ngAfterViewInit() (optional): Invoked after the component's view (including children) is initialized, suitable for DOM-related actions.
    • ngAfterViewChecked() (optional): Executes after the view and content are checked, suitable for tasks that rely on both view and content.
  3. Changes:
    • ngOnChanges() (optional): Triggers again when input properties change after the initial setup.
    • ngDoCheck() (optional): Runs after every Change Detection cycle, allowing custom logic or optimizations.
  4. Destruction:
    • ngOnDestroy(): Called before the component is destroyed, essential for cleaning up resources like subscriptions, timers, or external data connections.

Ogni componente in Angular segue un ciclo di vita ben definito, composto da varie fasi, dalla creazione alla distruzione.

Fasi principali:
  1. Creazione:
    • ngOnChanges() (opzionale): Si attiva quando le proprietà di input cambiano.
    • ngOnInit(): Viene invocato dopo che il componente riceve gli input iniziali e il contenuto è disponibile. Ideale per recuperare dati, impostare sottoscrizioni o eseguire calcoli.
    • ngDoCheck() (opzionale): Viene eseguito dopo ogni ciclo di rilevamento delle modifiche, consentendo una logica di rilevamento delle modifiche personalizzata.
  2. Rilevamento delle modifiche:
    • Questa fase prevede il rilevamento delle modifiche alle proprietà legate ai dati e l'aggiornamento della vista, se necessario.
    • ngAfterContentChecked() (opzionale): Richiamato dopo la verifica del contenuto, utile per attività come la manipolazione del DOM in base alle modifiche del contenuto.
    • ngAfterViewInit() (opzionale): Richiamato dopo l'inizializzazione della vista del componente (compresi i child), adatto per azioni legate al DOM.
    • ngAfterViewChecked() (opzionale): Viene eseguito dopo che la vista e il contenuto sono stati controllati, adatto per attività che si basano sia sulla vista che sul contenuto.
  3. Modifiche:
    • ngOnChanges() (opzionale): Si attiva nuovamente quando le proprietà dell'input cambiano dopo l'impostazione iniziale.
    • ngDoCheck() (opzionale): Viene eseguito dopo ogni ciclo di rilevamento delle modifiche, consentendo logiche o ottimizzazioni personalizzate.
  4. Distruzione:
    • ngOnDestroy(): Richiamato prima della distruzione del componente, essenziale per la pulizia di risorse come sottoscrizioni, timer o connessioni di dati esterni.
